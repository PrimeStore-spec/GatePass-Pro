⚙️ Admin Tools

🧹 Duplicate Entry Cleanup

Agar same entry do baar import ho gayi hai (CSV import bug ki wajah se), yahan se clean kar sakte ho.
Identical content wali entries (same GP/party/date/item/bags/weight/rate) ko duplicate maana jayega — sabse purani rakhi jayegi, baki delete.

🗄️ Legacy Backup Cleanup

Migration ke time humne purane app_data/records_*, journal_*, requests_* documents ko **safety backup** ke roop mein chhod diya tha.
Agar aap confident hain ki migration successful hai aur naya schema kaam kar raha hai, in legacy documents ko delete kar sakte ho.
⚠ Ek baar delete hone ke baad, naya schema hi raha — wapas nahi aa sakte.

☢️ Nuclear Wipe — Fresh Start (Current Firm)

Yeh sab kuch DELETE kar dega current firm ke liye:
  • Saari records (entries) — naye + legacy dono jagah
  • Saare payment journal entries
  • Saari pending/approved requests
  • Saare parties (Create Ledger)
  • Migration metadata
  • Local cache
⚠ User accounts, permissions, login — yeh sab safe rahega. Sirf data wipe hota hai.
📦 Backup zaroor download karo PEHLE! Restore karne ke liye chahiye hoga.

🔥 Force Sync (Emergency)

Use this if: App mein records dikhai nahi de rahe lekin Firestore Console mein abhi bhi hain. Ya delete karne ke baad bhi cloud mein zinda hain.
Kaam: Local data ko Firestore par mirror karega — Firestore mein extra (jo local mein nahi) records DELETE ho jayenge.
📦 Backup: Sab firms ka data ek JSON file mein download hoga (records, journal, requests, users, settings).
📂 Restore: Backup file load karke wapas restore karo. Existing data overwrite ho jayega — confirm prompt aayega.
